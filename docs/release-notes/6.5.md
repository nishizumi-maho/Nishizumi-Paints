# Nishizumi Paints 6.5

This is the first stable release since 6.0. It packages the showroom source controls, Teams downloads, Member ID downloads, all-paints exports, car dropdown UX, and the local 6.4.2 fixes into one public stable update.

## Highlights since 6.0

### Showroom Sources

- Advanced mode can choose which public Trading Paints Showroom sources are used by random fallback:
  - Trending
  - Newest
  - Most favorited
  - Most raced (ever)
- Multiple enabled sources are mixed randomly per fallback/download attempt.
- The default source selection is now all four sources.
- Logs include the source used for public Showroom fallback/downloads.

### Showroom Downloads

- Showroom now supports random public car downloads by choosing a car from the app's Trading Paints car list instead of typing TP car ID and slug values.
- Showroom links mode downloads one or more pasted Trading Paints paint links.
- Collection mode downloads usable public non-PRO paints from a Trading Paints collection and separates cars, helmets, and suits locally.
- Member ID mode downloads a member's car, helmet, and suit paints by Member ID and selected car.
- Member ID can also download all public paints for that member into an organized member folder.
- Teams mode downloads team car, helmet, and suit paints by Team ID and selected car.
- Teams can also scan mapped Trading Paints cars and download every found team asset into an organized team folder.
- Member ID and Teams all-paints actions now sit in a separate **Download everything** area so the full-download workflow is easier to find.

### Car Selection UX

- Car fields in Showroom Car, Member ID, and Teams are now searchable dropdowns.
- Clicking a car field opens the full car list.
- Typing filters the dropdown and shows concise suggestions below the field.
- Selecting a car closes the dropdown normally.
- The car field width and dropdown height were reduced for a cleaner layout.

### Local Random Pool

- **Recycle downloaded TP car paints into the local random pool** defaults to off for new/missing config values.
- Existing users who explicitly enabled that setting keep their preference.
- RandomPool size controls now support separate car, helmet, and suit caps.
- Manual Showroom downloads can target the standard RandomPool or a one-time custom folder.

### Session And Fallback Behavior

- Public Showroom fallback uses direct browserless downloads instead of relying on an embedded browser flow.
- Session totals are uncapped by default, with the protective cap behavior moved into Safe mode.
- Fallback handles car, helmet, and suit targets separately.
- Public Showroom fallback downloads spec maps when available.
- Collection fallback can run before normal public Showroom fallback and avoids repeating collection sources in the same session.
- Same-session roster refresh handles newly joined targets without reprocessing the whole session.

### App And Setup

- Easy mode exposes the core fallback controls for basic use.
- Advanced mode keeps the deeper source, pool, and fallback controls.
- Start with Windows/background behavior was cleaned up across installer and app settings.
- Quick Start and mapping review prompts were refined so first-time setup is less noisy.
- Showroom mapping tools can scan iRacing and Trading Paints public car lists, review missing mappings, open/edit the JSON mapping, reset the user mapping, and open the bundled seed.
