# Nishizumi Paints 6.9.1 Experimental

Draft hotfix for the 6.9 random controls build.

## Fixed

- `Restore previous` now clears the fixed override entry before restoring the previous paint.
- Team driver-personal liveries restored from history are registered as `driver_paint`, not `override`.
- `Forget` can fall back to the paint-history restore path when a Team target has no real Team Trading Paints asset but did have a driver-personal livery before the override.
- Team-driver preload no longer retries the same unavailable user/slot every poll; failed manifest/slot attempts and incomplete car-slot saves are throttled for the current session.
- Manual random override and restore actions are now guarded so the same selected driver item cannot launch duplicate concurrent jobs.
