# Nishizumi Paints 7.0

This release promotes the post-6.5 work into one public 7.0 release.

## Highlights

- Team sessions now support current-driver personal paint fallback for missing Team cars, helmets, and suits.
- Team driver swaps are confirmed before repainting and receive short follow-up refresh retries.
- The Session table now supports multi-select actions and double-click iRacing camera switching.
- Random fallback sources can be favorited or blocked from the Session tab.
- The app stores paint-history snapshots before manual overwrites and can restore the previous car, helmet, or suit.
- Restore previous now clears override metadata and reloads only affected cars.
- The Random tab layout is more compact, with Step 2 and Step 3 controls moved higher.
- New or missing configs protect the local user by default.

## Using the new Session controls

- `Ctrl+click` selects or deselects individual drivers.
- `Shift+click` selects a range.
- Compatible actions apply to all selected drivers that can use that action.
- Double-click a driver row to switch the in-game camera to that driver's car.
- Use `Favorite` or `Block` on random fallback sources to guide future random choices.
- Use `Restore previous` after a manual random or fixed override to restore the last saved local history snapshot.

## Team fallback order

1. Team paint for the item.
2. Current driver's personal paint for the item, if enabled.
3. Random fallback, if enabled and allowed for Team sessions.
4. iRacing default.

## Privacy and safety

- The normal public online fallback path is browserless.
- No Trading Paints password, token, cookie, or browser profile is required for public fallback.
- Runtime files such as settings, RandomPool content, collection caches, paint history, and iRacing paint folders should stay local.
