# Nishizumi Paints 7.2.0

Release date: 2026-06-20

## Automatic car identification

- Replaced the bundled and user-edited showroom mapping with the live Trading Paints template catalog.
- Combined Trading Paints vehicle MID/name/directory data with iRacing SDK `CarPath`, car name, and car ID.
- Recorded directories returned by real Trading Paints manifests as runtime validation.
- Added automatic alias learning when iRacing exposes a new directory for a uniquely matching vehicle name.
- Removed the seed JSON and the manual mapping-review workflow.

## Compatibility

- Normal session manifest downloads continue to use the directory supplied by iRacing and Trading Paints.
- Public showroom fallback, collections, Member ID tools, Team tools, and car selectors now read the automatic catalog.
- Manifest `<carid>` remains correctly treated as a paint asset ID rather than a Trading Paints vehicle MID.

## Validation

- Parser and identity regression tests cover template records, duplicate superspeedway variants, SDK alias learning, and manifest asset IDs.
- The release build is compiled from the published 7.1.3 baseline plus this change.
