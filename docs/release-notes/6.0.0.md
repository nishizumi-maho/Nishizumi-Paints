# Nishizumi Paints 6.0.0

## Highlights

- Browserless public-showroom fallback is now the main no-browser release path
- New Windows installer with in-place upgrade support
- New Easy mode with a simplified first-run experience
- Restored high-parallel Session Total mode without a hard cap
- Public showroom fallback now covers cars, helmets, and suits cleanly
- Tray icon now supports right-click menu actions and double-click restore

## Added

- Windows installer with upgrade support
- Easy mode
- UI mode selection in Quick Start
- AI member ID field in Quick Start
- Tray menu actions: **Open app** and **Exit**
- Double-click restore from the tray icon
- Easy-mode background-on-close toggle
- Manual public showroom downloads by car ID, direct links, and collection URL/ID

## Changed

- Public showroom direct downloads are the primary online fallback path in the no-browser release
- Session Total is now fully uncapped
- Safe mode now carries the protective caps and remains the recommended default
- Cars, helmets, and suits are processed together online by default again
- Random helmets and suits now default to enabled
- RandomPool recycling of downloaded session paints now defaults to disabled
- Advanced mode now labels autostart as **Start with Windows**
- Repository docs have been split into a concise README plus full wiki content

## Fixed

- Removed the extra delay between random online fallback downloads
- Fixed random session actions so they trigger iRacing texture reload immediately
- Fixed stale Session tab state across session changes
- Fixed noisy cleanup behavior during some session transitions
- Fixed public showroom fallback so helmets and suits work properly online
- Fixed online fallback spec maps so they download by default
- Fixed **Open TP** for fallback helmets and suits
- Fixed **Forget** so it restores the original TP paint when appropriate
- Fixed the bundled showroom mapping for BMW M2 CSR
- Fixed the installer/app startup-setting interaction
- Fixed the embedded-browser UI regression that referenced tkinter incorrectly

## Documentation and repository work

- Reworked the repository layout for release publishing
- Added wiki source pages for every major app area
- Added CI, CodeQL, Dependabot, and security policy files
